## 🏗️ 实现摘要

### 新增文件

#### 1. `src/app/api/decisions/route.ts`
**功能**: 待决策项聚合 API

**查询逻辑**:
- 状态为 todo 且有下一步行动的任务
- 优先级为 high 的任务
- 阻塞状态 (blocker=true) 的任务

**返回格式**:
```json
{
  "decisions": [
    {
      "id": 6,
      "title": "决策中心模块：待决策项聚合与一键上下文",
      "status": "blocked",
      "priority": "high",
      "owner": "main",
      "blocker": true,
      "next_action": "检查实现进度",
      "due_at": null,
      "updated_at": "2026-02-22T03:25:48.223Z",
      "source": "board"
    }
  ],
  "summary": {
    "total": 27,
    "highPriority": 26,
    "overdue": 0,
    "blocked": 3
  },
  "count": 27
}
```

#### 2. `src/components/DecisionCenter.tsx`
**功能**: 决策中心组件

**特性**:
- 显示待决策项列表（按优先级和阻塞状态排序）
- 每项可点击打开 DetailModal
- 悬停显示"一键复制上下文"按钮
- 统计信息徽章（总数/高优/阻塞/逾期）
- 支持手动刷新

**一键复制上下文格式**:
```
【待决策项 #6】
标题：决策中心模块：待决策项聚合与一键上下文
状态：blocked
优先级：high
负责人：main
下一步行动：检查实现进度
截止时间：—
更新时间：9 小时前
⚠️ 阻塞状态：是
来源：board
```

#### 3. `src/components/AlertCard.tsx`
**功能**: 告警卡片组件

**告警类型**:
- **阻塞告警** (error 级别): `blocked_count > 0`
- **待决策告警** (warning 级别): `pending_decisions > 0`
- **stale 告警** (warning/info 级别): 数据超过 2/24 小时未更新
- **Cron 失败告警** (error 级别): `cron_ok === false`

**工具函数**:
- `aggregateAlerts()`: 根据 health 和 tasks 数据自动生成告警列表
