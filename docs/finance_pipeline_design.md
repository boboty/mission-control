# 财务流程管线设计文档

## 业务背景

代理记账公司月度业务流程，支持多客户并行管理。

---

## 12 阶段完整流程

| 阶段 | 名称 | 分组 | 核心工作 | 负责人 |
|------|------|------|---------|--------|
| 1 | 客户对接 | 客户对接 | 签订合同、建立档案 | 客户经理 |
| 2 | 票据收集 | 票据收集 | 收取原始凭证 | 客户/会计助理 |
| 3 | 票据初审 | 票据收集 | 审核票据合规性 | 会计助理 |
| 4 | 银行对账 | 票据收集 | 核对银行流水 | 会计助理 |
| 5 | 凭证编制 | 账务处理 | 填制记账凭证 | 会计 |
| 6 | 账簿登记 | 账务处理 | 登记明细账/总账 | 会计 |
| 7 | 报表编制 | 账务处理 | 生成财务报表 | 会计 |
| 8 | 税务计算 | 税务申报 | 计算应纳税额 | 税务会计 |
| 9 | 纳税申报 | 税务申报 | 申报缴纳税款 | 税务会计 |
| 10 | 账务复核 | 复核归档 | 主管审核 | 财务主管 |
| 11 | 档案归档 | 复核归档 | 装订保管 | 档案员 |
| 12 | 客户交接 | 复核归档 | 交付资料、沟通税务 | 客户经理 |

---

## 阶段分组（看板列）

```
票据收集 (2-4)  →  账务处理 (5-7)  →  税务申报 (8-9)  →  复核归档 (10-12)
```

## 异常流转规则

| 异常 | 发生阶段 | 处理方式 | 流转 |
|------|---------|---------|------|
| 票据缺失 | 阶段 2-3 | 通知客户补正 | 2→2（循环） |
| 发票不合规 | 阶段 3 | 退回重开 | 2→2（循环） |
| 银行对账不平 | 阶段 4 | 编制调节表 | 暂停至完成 |
| 凭证错误 | 阶段 5-6 | 修正凭证 | 5→5（循环） |
| 税务计算差异 | 阶段 8 | 主管复核 | 8→8（循环） |
| 复核发现问题 | 阶段 10 | 退回修正 | 10→对应阶段 |

---

## 月度时间节点

```
每月 1-5 日   : 票据收集、票据初审 (阶段 2-3)
每月 6-10 日  : 银行对账、凭证编制、账簿登记 (阶段 4-6)
每月 11-15 日 : 报表编制、税务计算 (阶段 7-8)
每月 16-20 日 : 纳税申报 (阶段 9)
每月 21-25 日 : 账务复核、档案归档 (阶段 10-11)
每月 26-月末  : 客户交接、下月准备 (阶段 12→1)
```

## UI/UX 设计

### 看板视图
- 4 列：票据收集 / 账务处理 / 税务申报 / 复核归档
- 每列显示该组包含的阶段
- 卡片显示：公司名称 + 会计期间 + 当前阶段 + 状态

### 卡片状态
- 🟢 正常：按计划进行
- 🟡 警告：临近截止
- 🔴 阻塞：存在问题
- ⚪ 退回：等待补正

### 卡片操作
- 点击卡片 → 详情浮窗
- 拖拽卡片 → 阶段流转
- 右键菜单 → 快速操作（退回/阻塞/完成）

## API 设计

### GET /api/pipelines
```
参数：
- period: 会计期间 (2026-03)
- company_id: 客户 ID
- stage_group: 阶段分组
- status: 状态

返回：
{
  pipelines: [...],
  pagination: {...}
}
```

### PATCH /api/pipelines/:id/stage
```
入参：
{
  stage_no: 5,
  action: "advance" | "return",
  notes: "备注"
}

返回：
{
  success: true,
  pipeline: {...}
}
```

### POST /api/pipelines/:id/checklist
```
入参：
{
  stage_no: 3,
  checklist_items: [{name, done, note}]
}
```
