## ✅ T-20260221-023 数据完整性与空态策略

**目标**: 确保数据完整性，优化空态显示，提供 actionable 建议

### 改动摘要

1. **新增数据验证工具** `src/lib/data-validation.ts`
   - `validateTasks()`: 验证任务数据完整性
     - 检查必需字段（id, title）
     - 检查可选字段空值统计
     - 验证日期格式
     - 检查重复 ID
     - 验证枚举值（status, priority）
   - `validateHealthSnapshots()`: 验证健康快照数据
   - `generateDataQualityReport()`: 生成数据质量报告（调试用）

2. **增强 EmptyState 组件** `src/components/Card.tsx`
   - 新增 `moduleType` 属性，自动匹配建议操作
   - 新增 `suggestions` 属性，支持自定义建议
   - 为 6 个模块预设默认建议：
     - tasks: 创建任务/检查同步/确认配置
     - pipelines: 添加项目/检查同步/确认连接
     - events: 添加日程/检查配置/确认时区
     - memories: 创建归档/导入日志/检查路径
     - agents: 注册智能体/检查心跳/确认上报
     - health: 等待检测/检查 cron/查看日志

3. **更新页面空态** `src/app/page.tsx`
   - 所有空态调用添加 `moduleType` 属性
   - 集成数据验证逻辑
   - 新增数据质量警告提示条（仅在有 warnings 时显示）

## 🎯 决策问题（需要 main/用户确认）

### D-001: 指标趋势计算基准
**问题**: 当前趋势对比使用 health_snapshots 的历史数据，但该表主要记录阻塞数和待决策数，没有完整的任务统计。

**建议方案**:
- 方案 A: 在 health_snapshots 中增加 total/in_progress/blocked 字段，每次检测时记录
- 方案 B: 直接对比 24 小时前的 tasks 表快照（需要添加定时快照机制）
- 方案 C: 暂时不显示趋势，仅显示当前值（默认建议）

**推荐**: 方案 C（MVP 阶段优先保证功能可用，趋势功能可在后续迭代中添加）

### D-002: 导航项点击行为
**问题**: 当前导航项点击仅切换 activeModule 状态，没有实际页面跳转。

**建议方案**:
- 方案 A: 保持单页应用，通过 activeModule 控制显示内容（默认建议）
- 方案 B: 为每个模块创建独立路由（/tasks, /pipelines 等）

**推荐**: 方案 A（符合 MVP 架构，减少路由复杂度）

### D-003: 数据验证触发时机
**问题**: 当前数据验证在每次数据加载时运行，可能影响性能。

**建议方案**:
- 方案 A: 仅在开发环境运行验证（默认建议）
- 方案 B: 生产环境降低验证频率（如每 10 分钟一次）
- 方案 C: 仅在检测到数据异常时运行验证

**推荐**: 方案 A（开发阶段保证数据质量，生产环境信任 API 层）
